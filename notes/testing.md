# Testing

- Keep geometry, layout, scheduling, selection, and tile planning as pure functions where possible. They make cheap deterministic unit tests.
- `tests/fixtures/interaction.pdf` is generated by `scripts/create-test-pdf.rs` using only the standard library. It covers portrait, inherited Rotate 90, CropBox, RGB channel order, Unicode text, cross-page copy, and a nested four-entry outline.
- Backend tests load the pinned PDFium, render real tiles, inspect text, exercise cancellation/resume, and verify bounded high-resolution allocation.
- The vendor tile regression compares tiled cores with full-page output for portrait, rotation, CropBox, annotations, and AcroForm content.
- That regression must use a real multi-column 1024px-core grid. A small crop did not cross a tile boundary and missed rotated edge culling.
- Allow only tiny PDFium antialiasing differences at fractional scales; track differing channels, total delta, and maximum delta.
- Compare indexed text hit testing and visible-range queries with simple linear reference implementations on deterministic dense and adversarial inputs.
- Test large-document layout by comparing lazy geometry with the eager equations and checking that rescaling reuses shared geometry. Avoid timing assertions.
- The macOS E2E test opens the fixture, crosses minimum and maximum zoom while pages enter and leave the viewport, and requires a clean settled state before a hard timeout.
- The Classic feature E2E uses a temporary fixture copy, paints the real WYSIWYG editor, types and formats a multiword comment through native GPUI keystrokes, waits for autosave, returns with Escape, clicks the persisted highlight, opens it through both toolbar context and the comments list, creates all five colors, measures sidebar anchor drift, searches and navigates, then relaunches to verify sidecar persistence.
- The Fluid E2E types into the real editor, waits through the autosave debounce and sidecar worker, runs the production highlight mouse hit-test, exercises both comment-pane slide directions and list-to-editor navigation, checks Edit-note context resolution, and proves overlay width is added to horizontal scroll reach before reloading the sidecar.
- Theme coverage parses and applies every bundled config, checks required semantic tokens, distinct opaque dark paper, and icon assets, then launches native System, light, dark, and dark-with-light-PDF cases and requires a settled exact-tile QA report. Inspect one light Fluid and one dark Fluid window-only capture when changing token mappings or the PDF appearance toggle.
- TOC coverage uses the fixture's nested four-entry outline. The backend test verifies titles, hierarchy, page destinations, and explicit coordinate presence; pure tests verify centered stack geometry, triangular hover falloff, multiline callout sizing, breadcrumb indices, and largest-title matching. The native Fluid scenario crosses the delayed marker-to-callout bridge, resolves a page-only rotated-page entry by its heading text, requires the match to be centered, and settles with exact tiles.
- Test navigation emphasis as a deterministic state machine: it must wait for scroll settlement, replace an older target, cancel from both queued and active states, sanitize and cap PDF geometry, and expire after exactly 500 ms. Native navigation waits through that effect before reporting a settled viewport.
- Test text limits at the exact UTF-8 boundary and one scalar beyond it. Include native replacement ranges, IME composition, ignored search characters, and Markdown encoding overhead; every rejected edit must leave the full buffer state unchanged.
- Prove parser and rich-text traversal complexity with deterministic operation counts proportional to bytes, lines, and runs. Timing tests are noisy and can miss quadratic inputs at small fixture sizes.
- Inject a precise scroll packet while a sidebar transition is active. Waiting until after the slide misses frame-chain cancellation bugs that leave a panel partially open.
- Keep sidebar transition and search navigation phases separate when measuring anchor drift; result navigation intentionally moves the document.
- The debug `GPUI_PDF_READER_QA_*` hook dispatches real GPUI keystrokes and directly exercises the production command-wheel handler without Accessibility permission. Cocoa event translation and physical trackpad gestures remain manual checks.
- For the historical Metal freeze, require clean process exit and inspect app/system logs for invalid resources, GPU faults, and panics.
- Root tests do not run a dependency crate's own tests. Keep the focused vendored PDFium command in the saved suite.
- External-display regressions need a real macOS screen transition: move the native window between displays with different backing scales, then test external-display disconnect/reconnect. Resize-only simulation does not emit `windowDidChangeScreen` and cannot cover this GPUI platform path.
